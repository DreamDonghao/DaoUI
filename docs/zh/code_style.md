@page intro_page 代码风格规范

前言：统一的代码风格在项目开发中是一个十分重要又容易被忽视的部分。有一个统一的代码风格有助于开发人员更好地协作以及增强项目的可维护性。代码风格并没用一个严格的标准，而是重在统一。**如参与本项目的开发请务必遵循以下代码风格规定**。有不合理的地方欢迎联系项目作者。

## 基础规范

- 缩进：4个空格。

- 行宽限制：80

- 前花括号不换行

- 对于只有一条语句的代码块，也要用花括号包裹。

  ```C++
  if (val == 0) {
      val = 1;
  }
  ```

## 注释规范

统一使用Doxygen注释规范（///风格），对于定义的函数与类，必须给出注释。

## 命名规范

名词解释：

小驼峰命名法：第一个单词的首字母小写，后续单词的首字母大写。如`myValue`

大驼峰命名法：每个单词的首字母都大写。如`MyValue`

- 文件名：字母均小写，单词之间用下划线分隔。`my_class.hpp`
- 一般变量名：小驼峰命名法。`myValue`
- 函数名：小驼峰命名法。`myFunction`
- 类名：大驼峰命名法。`MyClass`

## 类定义结构规范

- 访问控制符顺序：public  protected  private
- 成员顺序优先级：类型别名 变量  构造函数 析构函数 赋值运算符 其他运算符 set函数 其他函数 get函数
- 所有成员变量皆以m_开头
